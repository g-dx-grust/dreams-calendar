# 04. Claude Code向け初期プロンプト集

このドキュメントは、Claude Code（またはその他のAIエージェント）に実装を指示する際にそのままコピー＆ペーストして使用できるプロンプト集です。

## 【Phase 1】初期セットアップとDBマイグレーション

最初のタスクとして、環境構築とデータベース設計を指示します。

```text
あなたは株式会社グラストのエンジニアとして、G-DXプロジェクトの新規システムを開発します。
システム名は「G-DX For スケジュール」です。

【必須：作業前に読むこと】
1. `CLAUDE.md`（グラスト共通開発ルール）を読み込み、厳守してください。
2. 既存の `kanri-system`（案件管理）の `DESIGN.md` および `G-DX_Design_System.md` を読み込み、デザインルールを厳守してください。

【目的】
Lark標準カレンダーの課題（複数人の予定が俯瞰しづらい）を解決する、サイボウズ風の独自カレンダーWebアプリを構築します。

【技術スタック】
- Next.js 15（App Router）
- TypeScript（strict mode）
- Tailwind CSS v4（dreaMsデザイントークン適用）
- shadcn/ui（カスタマイズ）
- Supabase（kanri-systemと同一プロジェクトを想定）
- date-fns（日時処理）

【今回のタスク】
まずはデータベースの設計とマイグレーションファイルの作成を行います。
提供されている `02_database_schema.md` を読み込み、`schedules` テーブルと `schedule_types` テーブルの Supabase マイグレーションSQLを作成してください。
また、TypeScriptの型定義ファイルも生成・整備してください。
```

## 【Phase 1】日表示カレンダーの実装（最優先）

データベースの準備ができたら、最重要機能である「日表示カレンダー」の実装を指示します。

```text
続いて、最優先機能である「日表示カレンダー」のUIコンポーネントを実装します。

【実装要件】
1. 縦軸=社員（人別行）、横軸=時間（8:00〜18:00、30分刻み）のタイムラインレイアウトをCSS Grid + 絶対配置で実装してください。社員列（左端）は `position: sticky` で固定表示します。
2. 予定ブロックは、`schedule_types` の色（例: `#3370FF`）を背景色として表示します。
3. 予定ブロック内には「開始〜終了時刻」「件名」「案件番号」を表示してください。
4. `dnd-kit` を導入し、予定ブロックをドラッグ＆ドロップで別の時間帯や別の担当者の行へ移動できるようにしてください。
5. 画面上部に、表示する社員をチェックボックスで切り替えられるフィルター機能を実装してください。

【デザイン厳守事項】
- 角丸は最大6px（`rounded-md`）まで。8px以上は禁止。
- アクセントカラーは `#3370FF` のみ。
- グラデーション・パステルカラー・絵文字の使用禁止。
- 影（shadow）は原則使わず、1pxのボーダー（`#DEE0E3`）で領域を区切る。

まずはモックデータを使用して、グリッドの描画とドラッグ＆ドロップが動作するコンポーネントを作成してください。
```

## 【Phase 1】予定登録フォームと案件検索

カレンダーUIができたら、予定の登録機能と案件管理システムとの連携（検索）を実装します。

```text
次に、予定の登録・編集フォームを実装します。

【実装要件】
1. `react-hook-form` と `zod` を使用してバリデーション付きのフォームを実装してください。
2. 入力項目：タイトル、担当者、開始日時、終了日時、予定種別、案件番号、場所、メモ。
3. 「案件番号」入力欄は、`03_api_integration.md` に記載の通り、`kanri-system` のAPI（またはモック関数）を呼び出してサジェスト検索できるようにしてください。
4. フォームの送信は Server Actions パターン（`"use server"`）で実装してください。
5. 楽観的UIを優先し、保存ボタン押下時は即座にスピナー（`loading` 状態）を表示してください。

実装が完了したら、実際に予定を登録し、カレンダー上に表示されることを確認してください。
```

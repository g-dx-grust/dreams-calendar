# 01. 実装ガイド

本ドキュメントは、カレンダーWebアプリの実装スコープ、画面仕様、およびデザイン原則を定義します。

## 1. 実装フェーズ

### Phase 1（MVP）— まず動くカレンダーを作る

**目標:** 社員が「今のサイボウズと同じ感覚で使える」カレンダーを完成させる。入力項目を増やしすぎないことを最優先とします。

| 優先度 | 機能 | 詳細 |
|---|---|---|
| **最優先** | 日表示（人別行） | 縦軸=社員、横軸=時間（8:00〜18:00、30分刻み）のグリッド |
| **最優先** | 予定種別ごとの色分け | 予定ブロックの背景色を種別で変える |
| **最優先** | ドラッグ＆ドロップ | 時間移動・担当者変更・別日移動 |
| **最優先** | 予定詳細画面 | ポップアップではなく専用ページ or 詳細パネル |
| 高 | 予定登録・編集・削除 | フォームに案件番号入力欄を含む。終日チェックボックスで複数日終日予定を作成可能（終日時は 00:00〜23:59） |
| 高 | 週表示・月表示 | 日表示の次に実装 |
| 高 | 担当者フィルター | チェックボックスで表示社員を切替 |
| 中 | 社員マスタ・予定種別マスタ | 管理画面で色設定可能 |
| 中 | スマホ簡易表示 | 今日の自分の予定のみ（完了ボタンは廃止） |
| 中 | 日報ボタン（日ビュー） | 各社員行の「社員名」と「時間軸」の間に「日報」ボタンを配置。クリックでモーダルを開き、本文を書いて提出する。提出後はチェックマーク表示に切替（再度開いて編集可能）。完了済み予定は日報内に自動表示し、提出時は管理者へ Lark 通知を送信 |
| 中 | 直接完了 | 予定詳細ポップオーバーから「完了」を押せる。ただしワンクリック完了にはせず、作業時間（実施時間）の終了時刻と作業メモを確認してから確定する |
| 中 | オンライン予定 | 予定種別に「オンライン」を用意し、選択時に会議URLを自動発行して予定詳細に表示する。Lark会議連携への切替を前提にする |

### Phase 2 — 案件管理との連携強化

- 案件番号サジェスト検索（`kanri-system` の `cases` テーブルを参照）
- 予定詳細から案件詳細画面（`/cases/[id]`）へのリンク
- 案件別予定一覧・作業時間集計
- 完了済み予定からの日報自動生成の精度向上

### Phase 3 — 作業時間・採算・拡張機能

- 作業時間（実施時間）の分析と標準時間見直し
- 採算管理（売上 vs 稼働時間 × 時間単価）
- 車両・会議室予約
- 通知機能

## 2. 画面仕様

### 2.1 日表示（最重要画面）

```text
2026年5月3日（日）
社員     08:00   09:00   10:00   11:00   12:00   13:00   14:00   15:00   16:00   17:00   18:00
山田     [現場 2026-LI-001 .........][移動][............ 現場 2026-LI-001 .................]
鈴木             [役所 2026-FC-002 ........]              [書類作成 ............]
佐藤     [社内 2026-BS-003 ......][来客 ......]
田中             [測量 2026-LI-004 .........................]
高橋                                              [重要：役員会 ..............]
```

- 縦軸=社員（人別行）／横軸=時間（30分刻みのタイムライン）
- 横スクロールで時間帯を、縦スクロールで社員数を吸収
- 社員列（左端）は `position: sticky` で固定表示
- 表示する社員はチェックボックスで切替
- 予定ブロックには「開始〜終了時刻」「件名」「案件番号」「予定種別」を表示

### 2.2 予定登録フォーム

```text
予定登録
タイトル     [　　　　　　　　　　　]
担当者       [山田 ▼]
同行者       [鈴木 ▼]（複数選択可）
開始日時     [2026/05/03 08:40]
終了日時     [2026/05/03 17:00]
予定種別     [現場 ▼]
案件番号     [2026-LI-001 ▼]（サジェスト検索）
案件名       [豊橋市A現場]（案件番号選択で自動入力）
場所         [　　　　　　　　　　　]
メモ         [　　　　　　　　　　　]
[登録する] [キャンセル]
```

### 2.3 予定詳細画面

予定クリックで専用ページまたは詳細パネルに切替（ポップアップ禁止）。
表示項目：件名・時間（開始日時〜終了日時を 1 行に統合。同日は日付を 1 回のみ、別日にまたがる場合は開始日と終了日を両方表示）・担当者・同行者・予定種別・案件番号・案件名・現場名・顧客名・場所・メモ・完了ステータス・関連案件リンク・作業時間（実施時間）・会議URL。

### 2.4 完了と日報

- 予定カードまたは予定詳細から「完了」を押せる。ただし即時完了にはせず、作業終了時刻を確認・修正してから確定する。
- 作業時間（実施時間）は `actual_start_at` / `actual_end_at` / `actual_minutes` として保存し、当初予定の `start_at` / `end_at` は上書きしない。
- 完了時に作業メモを登録できる。日報画面では完了済み予定だけを自動表示し、作業時間（実施時間）と作業メモを確認できる。
- 日報上の作業時間（実施時間）は、予定より早いまたは予定どおりなら青、予定より遅いなら赤の文字色で表示する。背景色は変更しない。
- 本人アカウントは自分の日報のみ表示する。管理者（`users.role = 'admin'`）は全員の日報を閲覧し、コメントできる。
- 日報提出時は Lark に通知する。通知のボタンから `/calendar?view=day&date=YYYY-MM-DD#daily-report-USER_ID` に遷移し、該当日報へ直接移動できる。

## 3. デザイン原則（dreaMsデザインルール準拠）

`kanri-system` と一貫性を持たせるため、以下のデザインルールを厳守します。

- **カラー:** 
  - Primary（メインカラー）は `#3370FF`（Lark Blue）のみ。
  - ヘッダーは濃色ではなく「白背景＋下境界線1px」。
  - 画面の95%は白・グレー・黒で構成し、装飾目的の色は使わない。
- **角丸:** `4px` または `6px` のみ（最大 `rounded-md`）。`8px`以上は使用禁止。
- **タイポグラフィ:** Noto Sans JP を使用。ベースサイズは `15px`。
- **影（Shadow）:** 原則使用禁止。領域は `1px` のボーダー（`#DEE0E3`）で区切る。
- **UX:** 
  - 楽観的UIを優先し、保存ボタン押下時は即座にスピナー表示（最低200ms）。
  - 過度なアニメーションやトーストの多発は避ける。

## 4. カレンダーUIライブラリの選択指針

**独自実装（推奨）:** 
サイボウズ風の「人別行×時間軸」タイムライン（縦=社員 / 横=時間）は FullCalendar の標準機能では実現困難です。CSS Grid + 絶対配置 + `dnd-kit` で独自実装する方が、カスタマイズの自由度と保守性が高くなります。

**FullCalendar を使う場合:** 
`resourceTimelineView` または `resourceTimeGridView` を利用しますが、大幅なカスタマイズが必要になります。FullCalendar のライセンス（有料プラン）にも注意が必要です。

# サイボウズ風カレンダーWebアプリ 開発ドキュメント

本ディレクトリは、Lark標準カレンダーの課題を解決し、案件管理システム（`kanri-system`）と連携する「サイボウズ風カレンダーWebアプリ」を開発するためのドキュメント一式です。

Claude Code（またはその他のAIエージェント）と開発者が伴走して実装を進めるための「設計の地図」として機能します。

## ドキュメント構成

| ファイル名 | 内容 | 用途 |
|---|---|---|
| `01_implementation_guide.md` | 実装ガイド（フェーズ、画面仕様、連携方針） | 開発の全体像とマイルストーンの確認 |
| `02_database_schema.md` | データベース設計（テーブル定義、初期データ） | Supabaseのマイグレーション作成 |
| `03_api_integration.md` | 案件管理システム（`kanri-system`）との連携仕様 | 案件検索や実績時間連携の実装 |
| `04_claude_prompts.md` | Claude Code向け初期プロンプト集 | 実装開始時にClaude Codeに渡す指示書 |
| `CLAUDE.md` | カレンダーシステム用AIエージェント行動規範 | プロジェクト直下に配置するルールブック |

## プロジェクトの目的とコンセプト

**目的:** 
既存のLarkカレンダーでは複数人の予定が俯瞰しづらいという現場課題を解決するため、**サイボウズのグループウェアカレンダーに近い視認性と操作性を持つ独自カレンダーWebアプリ**を開発します。

**最重要コンセプト:**
- 縦軸を社員、横軸を時間（30分刻み）とする「グループ日表示（人別行 × 時間タイムライン）」が最優先
- 予定の色分けは「担当者」ではなく「予定種別（現場・社内・役所など）」で行う
- ドラッグ＆ドロップで予定を別時間帯・別担当者へ直感的に移動できる
- 将来的には案件管理・日報・稼働集計・採算管理の入口になる設計にする

## アーキテクチャ方針

- **フレームワーク:** Next.js 15 (App Router) + React 19
- **スタイリング:** Tailwind CSS v4 + shadcn/ui (dreaMsデザインルール準拠)
- **データベース:** Supabase (PostgreSQL) ※`kanri-system` と同一プロジェクトを共有
- **カレンダーUI:** 独自実装（CSS Grid + dnd-kit）推奨

## 開発の進め方

1. まず `CLAUDE.md` をプロジェクト直下に配置してください。
2. `04_claude_prompts.md` に記載されているプロンプトを Claude Code に渡し、実装を開始してください。
3. 疑問点や仕様の齟齬が生じた場合は、各ドキュメントを参照して方針を決定してください。
